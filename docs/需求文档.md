# Markdown 转 Word/PDF 工具 - 需求文档

## 一、项目概述

### 1.1 项目名称

Markdown 转 Word/PDF 工具（md2doc）

### 1.2 项目目标

开发一个**纯前端**的 Markdown 转换工具，支持将 Markdown 内容转换为 Word (.docx) 和 PDF 格式，并支持多种可切换、可编辑的样式模板。

### 1.3 目标用户

- 需要将 Markdown 文档导出为正式文档的写作者
- 需要统一文档样式的团队
- 希望快速生成多格式文档的用户

## 二、功能需求

### 2.1 核心功能

#### F1：Markdown 编辑与预览

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| F1.1 | 提供 Markdown 输入区域，支持粘贴和编辑 | P0 |
| F1.2 | 支持实时预览，输入后即时渲染 | P0 |
| F1.3 | 支持常见 Markdown 语法（标题、列表、表格、代码块、引用、链接、图片等） | P0 |
| F1.4 | 支持代码块语法高亮（可选） | P1 |
| F1.5 | 编辑器提供粘贴和清除快捷操作按钮 | P1 |

#### F2：导出功能

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| F2.1 | 支持导出为 Word (.docx) 格式 | P0 |
| F2.2 | 支持导出为 PDF 格式 | P0 |
| F2.3 | 导出时应用当前选中的样式模板 | P0 |
| F2.4 | 导出前可自定义文件名（可选） | P2 |
| F2.5 | 样式栏底部和预览栏头部均提供导出入口，功能一致 | P0 |

#### F3：样式模板系统

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| F3.1 | 提供至少 3 个预设模板（如：默认、学术、技术文档） | P0 |
| F3.2 | 支持在模板之间切换 | P0 |
| F3.3 | 切换模板时，预览和导出均使用新模板样式 | P0 |
| F3.4 | 展示当前模板的样式配置（可读形式） | P0 |
| F3.5 | 支持修改模板的样式配置，并实时反映到预览 | P0 |
| F3.6 | 支持"自定义"模板，用户修改后自动保存到本地（localStorage），刷新页面后保留 | P0 |

#### F5：预览增强

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| F5.1 | 预览区支持缩放控制（放大 / 缩小 / 显示百分比） | P1 |
| F5.2 | 预览以纸张形式展示，带阴影效果，所见即所得 | P0 |

**模板持久化规则：**

- **预设模板**（默认、学术、技术文档）：内置只读，用户可在界面上临时调整样式并实时预览/导出，但刷新页面后恢复为预设默认值，不做本地持久化
- **自定义模板**：提供一个独立的"自定义"模板槽位，用户对其所做的所有样式修改自动保存到 localStorage，刷新页面后自动加载上次保存的配置

## 三、界面需求

### 3.1 整体布局

采用**顶栏 + 三栏**布局：

**顶栏（Top Bar）**

| 元素 | 说明 |
|------|------|
| Logo 区域 | 左侧展示应用图标（file-text）、名称「md2doc」及副标题「Markdown → Word / PDF」 |
| 主题切换按钮 | 右侧图标按钮，切换明暗主题（P2） |
| GitHub 链接 | 右侧图标按钮，跳转项目仓库（P2） |

**三栏区域**，从左到右依次为：

| 栏位 | 宽度建议 | 内容 |
|------|----------|------|
| 第一栏（样式栏） | 固定 300px | 模板选择、样式配置表单、导出按钮 |
| 第二栏（编辑栏） | 可伸缩，填充剩余空间 | Markdown 输入/粘贴区域 |
| 第三栏（预览栏） | 可伸缩，填充剩余空间 | 实时渲染的预览区域 |

### 3.2 第一栏：样式栏

- **模板选择**：卡片列表形式，每张卡片包含图标和模板名称，选中卡片高亮（主题色背景 + 白色文字）
  - 默认模板（file-text 图标）
  - 学术论文（graduation-cap 图标）
  - 技术文档（code 图标）
  - 自定义（palette 图标）
- **样式配置**：根据当前模板展示可编辑的配置项（表单形式），各配置区块用分隔线隔开
- **导出区域**：底部提供「导出 Word」（主色实心按钮）和「导出 PDF」（描边按钮）两个全宽按钮
- 配置修改后，预览区应实时更新

### 3.3 第二栏：编辑栏

- **编辑器头部**：左侧显示编辑图标和「Markdown 编辑」标题，右侧提供「粘贴」和「清除」操作图标按钮
- 多行文本输入框，使用等宽字体（如 JetBrains Mono）
- 支持滚动
- 可提供占位提示文字（如「在此粘贴或输入 Markdown...」）

### 3.4 第三栏：预览栏

- **预览头部**：左侧显示预览图标和「实时预览」标题，右侧提供缩放控制（- 100% +）及快捷导出按钮（PDF / Word）
- 预览区渲染 Markdown 为 HTML，并应用当前模板样式，以纸张（Page）形式展示，带阴影效果
- 支持滚动，与编辑区独立
- 样式与导出结果保持一致（所见即所得）

### 3.5 响应式（可选）

- 小屏幕下可改为上下堆叠或标签切换
- 优先级：P2

## 四、非功能需求

### 4.1 技术约束

| 需求 | 说明 |
|------|------|
| NF1 | **纯前端**：所有逻辑在浏览器内完成，无需后端服务 |
| NF2 | 不依赖网络（除首次加载资源外），支持离线使用 |
| NF3 | 不上传用户内容到服务器，保护隐私 |

### 4.2 性能

| 需求 | 说明 |
|------|------|
| NF4 | 预览响应延迟 < 300ms（常规文档） |
| NF5 | 导出 10 页以内文档在 5 秒内完成 |

### 4.3 兼容性

| 需求 | 说明 |
|------|------|
| NF6 | 支持 Chrome、Edge、Firefox、Safari 最新两个主版本 |
| NF7 | 支持中文内容及常用中文字体 |

### 4.4 可用性

| 需求 | 说明 |
|------|------|
| NF8 | 界面简洁，主要操作在 3 步内完成 |
| NF9 | 导出失败时给出明确提示 |

## 五、预设模板示例

### 5.1 模板一：默认

- 正文字体：微软雅黑 / 无衬线
- 正文字号：12pt
- 正文段落间距：6pt
- 标题：`#` 22pt 加粗左对齐，`##` 18pt 加粗，`###` 15pt 加粗，`####` 13pt 加粗
- 标题上下间距：`#` 12pt/6pt，`##` 10pt/4pt，`###` 8pt/4pt，`####` 6pt/2pt
- 行高：1.5
- 表格：全部边框，表头浅灰背景加粗
- 分隔线：实线 #cccccc 1px
- 页边距：25mm

### 5.2 模板二：学术论文

- 正文字体：Times New Roman / 宋体
- 正文字号：12pt
- 正文段落间距：0.5em
- 标题：`#` 22pt 加粗居中，`##` 18pt 加粗，`###` 14pt 加粗，`####` 13pt 加粗
- 标题上下间距：`#` 1em/0.5em，`##` 0.8em/0.4em，`###` 0.6em/0.3em，`####` 0.5em/0.25em
- 行高：1.5 或 2
- 表格：仅横线，表头加粗
- 分隔线：实线 #999999
- 页边距：25.4mm（1 英寸）

### 5.3 模板三：技术文档

- 正文字体：系统无衬线
- 正文字号：11pt
- 正文段落间距：4pt
- 标题：`#` 20pt 加粗左对齐，`##` 16pt 加粗，`###` 14pt 加粗，`####` 13pt 加粗
- 标题上下间距：`#` 10pt/4pt，`##` 8pt/4pt，`###` 6pt/3pt，`####` 5pt/2pt
- 代码块：Consolas 11pt，浅灰背景，有边框
- 行高：1.4
- 表格：全部边框，表头 #e8e8e8 背景加粗
- 分隔线：虚线 #dddddd
- 页边距：20mm

## 六、数据流说明

```
用户输入 MD
    ↓
[样式配置] + [MD 文本]
    ↓
markdown-it 解析 → HTML
    ↓
应用 CSS（来自模板配置）
    ↓
├─→ 预览区渲染
├─→ html2pdf.js → PDF 下载
└─→ remark-docx → DOCX 下载
```

### 7.1 基本验收

- [ ] 顶栏 + 三栏布局正确展示
