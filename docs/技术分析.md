# Markdown 转 Word/PDF 工具 - 技术分析

## 一、技术选型概览

### 1.1 核心能力需求

| 能力 | 技术方案 | 说明 |
|------|----------|------|
| Markdown 解析 | markdown-it | 主流、可扩展、支持 GFM |
| Markdown → HTML | markdown-it.render() | 用于预览和 PDF 生成 |
| Markdown → Word | remark-docx / docx | 纯前端生成 .docx |
| HTML → PDF | html2pdf.js / jsPDF | 纯前端生成 PDF |
| 样式模板 | JSON 配置 + CSS 变量 | 可序列化、可切换 |

---

## 二、Markdown 解析层

### 2.1 推荐：markdown-it

- **优势**：CommonMark 兼容、插件丰富、浏览器可用、约 19M 周下载
- **插件**：markdown-it-anchor、markdown-it-highlightjs（代码高亮）
- **输出**：HTML 字符串，可直接用于预览和 PDF 流程

### 2.2 备选：marked

- 更轻量，API 简单，但扩展性不如 markdown-it

---

## 三、Markdown → Word (.docx)

### 3.1 方案 A：remark-docx（unified 生态）

- **原理**：Markdown → MDAST → docx 库 → .docx 文件
- **特点**：支持表格、数学公式、图片、代码块
- **样式**：通过 docx 库的 Run/Paragraph 配置控制字体、字号、颜色等
- **适用**：需要较完整 Markdown 支持时

### 3.2 方案 B：@mohtasham/md-to-docx

- **特点**：TypeScript、支持目录、页码、可配置样式
- **周下载**：约 11.7K
- **适用**：需要开箱即用的样式配置时

### 3.3 方案 C：MD → HTML → docx（docshift 思路）

- **原理**：先转 HTML，再用 HTML→DOCX 库
- **优点**：预览和导出可共用同一套 HTML/CSS
- **缺点**：HTML→DOCX 的样式映射可能不如原生 docx 精细

### 3.4 推荐

优先考虑 **remark-docx** 或 **@mohtasham/md-to-docx**，直接生成 docx，样式可控性更好。若模板系统以 CSS 为主，可评估 HTML→DOCX 方案。

---

## 四、Markdown → PDF

###  方案 ：html2pdf.js

- **原理**：HTML → html2canvas → jsPDF
- **优点**：纯前端、易用、支持分页
- **缺点**：大文档时 canvas 渲染可能较慢


---

## 五、样式模板系统

### 5.1 模板数据结构（JSON）

```json
{
  "id": "template-1",
  "name": "学术论文",
  "styles": {
    "fontFamily": "Times New Roman",
    "fontSize": 12,
    "heading1": { "fontSize": 18, "bold": true },
    "heading2": { "fontSize": 16, "bold": true },
    "code": { "fontFamily": "Consolas", "backgroundColor": "#f5f5f5" },
    "lineHeight": 1.5,
    "margin": { "top": 72, "bottom": 72, "left": 72, "right": 72 }
  }
}
```

### 5.2 实现方式

- **预览**：将 `styles` 映射为 CSS 变量或内联样式，应用到渲染后的 HTML
- **Word**：将 `styles` 映射为 docx 的 RunFormatting、ParagraphFormat 等
- **PDF**：与预览一致，使用同一套 CSS

### 5.3 模板切换

- 预设 3–5 个模板（如：默认、学术、技术文档、简洁）
- 切换时加载对应 JSON，更新预览和导出逻辑
- 支持在界面上修改配置并实时预览

## 六、三栏布局技术

### 6.1 布局结构

```
┌─────────────┬──────────────────┬─────────────────┐
│   样式栏    │    Markdown 栏   │    预览栏       │
│  (约 280px) │    (可伸缩)      │   (可伸缩)      │
│             │                  │                 │
│ - 模板选择  │  - 粘贴/编辑     │  - 实时渲染     │
│ - 样式配置  │  - 支持滚动      │  - 样式应用     │
│ - 导出按钮  │                  │  - 滚动同步     │
└─────────────┴──────────────────┴─────────────────┘
```

### 6.2 技术实现

- **框架**：React / Vue / 原生 JS 均可
- **布局**：CSS Grid 或 Flexbox，支持中间栏拖拽调整宽度
- **预览**：`iframe` 或 `contenteditable` 的 `dangerouslySetInnerHTML`，配合沙箱样式隔离

## 七、技术栈建议

|------|----------|------|

| 状态管理 | Zustand 或 React Context | - |

## 八、潜在问题与对策

| 问题 | 对策 |
|------|------|
| 大文档导出慢 | 分块渲染、进度提示、考虑 Web Worker |
| 图片跨域 | 需转为 base64 或同源；可提示用户使用本地/同源图片 |
| 中文字体 | 确保系统或引入字体支持中文，PDF/Word 中正确嵌入 |
| 样式一致性 | 预览、Word、PDF 共用同一套「样式配置 → 渲染」映射逻辑 |
| 浏览器兼容 | 目标现代浏览器（Chrome/Edge/Firefox/Safari 近期版本） |

## 十、总结

- **纯前端**：markdown-it + remark-docx + html2pdf.js 可完全在浏览器内完成 MD → Word/PDF
- **模板系统**：用 JSON 描述样式，分别映射到 CSS（预览/PDF）和 docx API（Word）
- **三栏布局**：Grid/Flex + 可调宽度，左样式、中编辑、右预览
- **实现路径**：先完成 MD 解析与预览，再接入 Word/PDF 导出，最后完善模板与样式配置
